Проверка интеграции Google OAuth в системе Tire Service

☁️ Этап 1: Настройка Google Cloud Console

⚠️ Важно: Для полного тестирования требуется настройка Google Cloud Console

1.1 Создание проекта в Google Cloud Console

  • Перейти в Google Cloud Console
  • Создать новый проект "Tire Service OAuth"
  • Включить Google+ API

1.2 Настройка OAuth 2.0

  • Перейти в "APIs & Services" → "Credentials"
  • Создать OAuth 2.0 Client ID
  • Добавить Authorized redirect URIs:
    http://localhost:8000/auth/google/callback
    http://localhost:3008/auth/google/callback

1.3 Получение учетных данных

  • Скопировать Client ID и Client Secret
  • Добавить в .env файл бэкенда
# tire-service-master-api/.env GOOGLE_CLIENT_ID=your_client_id_here GOOGLE_CLIENT_SECRET=your_client_secret_here FRONTEND_URL=http://localhost:3008

🔄 Этап 4: Полное тестирование процесса

4.1 Тестирование входа через Google

  • Клик на "Войти через Google" перенаправляет на Google
  • После авторизации создается/находится пользователь
  • JWT токен устанавливается в cookies
  • Перенаправление на правильную страницу по роли

4.2 Тестирование привязки аккаунта

  • Авторизация обычным способом
  • Переход в настройки профиля
  • Клик "Привязать Google аккаунт"
  • Успешная привязка и возврат в настройки

4.3 Тестирование отвязки аккаунта

  • Клик "Отвязать Google аккаунт"
  • Подтверждение в диалоге
  • Успешная отвязка и обновление UI

🔒 Этап 5: Проверка безопасности

5.1 Валидация данных

  • Проверка уникальности google_id
  • Валидация email от Google
  • Защита от CSRF атак

5.2 Авторизация

  • JWT токены работают корректно
  • Cookies устанавливаются безопасно
  • Роли пользователей определяются правильно
